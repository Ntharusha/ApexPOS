# ApexPOS Architecture & Functional Map

This document is designed to give you (or your UI tool) a complete picture of how the **ApexPOS React Application** works, including all pages, components, features, and the global state management. Use this to safely reshape and re-theme the UI without breaking functionality.

## 1. Global State & Styling Mechanism

- **State Manager:** Zustand (`client/src/store/useStore.ts`)
- **Styling Architecture:** TailwindCSS with a custom dynamic theme (Dark/Light mode). Most containers heavily utilize glassmorphism (via classes like `bg-white/5` or `bg-surface/50` with `backdrop-blur`).
- **Icons:** `lucide-react`
- **Animations:** `framer-motion` (used on almost every button click, modal appearance, and page transition).

### Core Zustand State (`useStore()`)
- **Theme/Layout:** `theme` ('light' | 'dark'), `sidebarOpen`, `toggleSidebar()`, `setTheme()`
- **Auth & User:** `isAuthenticated`, `user` (id, name, role, branch_id), `token`, `login()`, `logout()`
- **Cart & POS:** `cart` (Array of items), `addToCart()`, `removeFromCart()`, `updateQuantity()`, `clearCart()`, `currentShift`
- **Offline/Sync Engine:** `pendingSales`, `isOnline`, `queueSale()`, `offlineProducts`, `offlineCategories`, `syncStatus`, `syncOfflineData()`

## 3. Key Components (`client/src/components/`)

### Layout
- **`Layout.tsx`**: Wraps authenticated routes. Contains the `Sidebar`, `Header`, and an `AnimatePresence` page transition boundary.
- **`Header.tsx`**: Contains the `ShiftModal` trigger, the Theme toggle switch, notifications bell, and global User Profile dropdown.
- **`Sidebar.tsx`**: Dynamic routing menu. Renders specific links based on the `user.role`.
- **`SyncEngine.tsx`**: Headless component that listens to `window.addEventListener('online')` to sync queue sales from IndexedDB/localStorage.

### Common Utilities
- **`ShiftModal.tsx`**: Mandatory modal for cashiers. Prompts for "Opening Float" (cash in drawer) before allowing access to `RetailPOS`.
- **`Receipt.tsx`**: Hidden/Printable DOM node. Formatted perfectly for 80mm thermal receipt printers. Uses the browser `window.print()` JS API.
- **`SyncStatus.tsx`**: Floating pill at the bottom indicating whether the app is Online or operating entirely in Offline Mode.
- **`AnimatedIcon.tsx`**: Helper utility to add pulsing/bouncing micro-interactions on hover.

### POS & Complex Modals
- **`CheckoutModal.tsx`** (Inside `pos/`): The final flow triggered by RetailPOS. Handles Payment Method selection (Cash, Card, Split, LANKA_QR). Allows input of tendered amounts and calculates change owed.
- **`CustomerCRMPanel.tsx`** (Inside `pos/`): A slide-out panel that pulls customer loyalty profiles, previous order histories, and suggests AI predictive upsells based on the selected patron.

## 4. Theme & Hierarchy Modifications Guide

If you insert this code into an intelligent UI builder (or LLM) to apply a theme, make sure it understands the following CSS rules you have established:

1. **`glass-card`**: A custom global class. You can usually safely alter its definition in `index.css` to instantly change the vibe of all widgets.
2. **Foreground/Background Tokens**: Usage of `bg-surface`, `bg-primary`, `text-text`, and `text-text-muted` allows you to change the global theme variables in your main config, while preserving structural legibility.
3. **Transition properties**: You heavily utilize `transition-all` on hover states.

**To successfully apply a new layout or theme, tell your builder:**
> *"Preserve all `onClick`, `zustand state bindings`, and `framer-motion` properties. Only change the Tailwind CSS classes, spacing, and structural HTML wrappers to match the newly requested design system."*
